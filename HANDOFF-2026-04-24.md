## The block-7 calibration event

This is the load-bearing finding to internalize before doing anything
else:

- Block 3 (N=5) reported structural-at-500k beat STDP-only by +25% at
  +1.01e-04 vs +7.96e-05. Looked solid at the time.
- Block 7 (N=20) reports +3.6% effect (+7.75e-05 vs +7.47e-05), z=0.53
  — clearly null.
- **The N=5 point estimate was ~7× larger than the truth.** Not wider
  CIs around the same number — just wrong.

Implication: any prior claim in steps 1–17 measured at N<20 is suspect.
Most of the project's headline numbers are N=1 or N=5.

below). Copy it onto the USB stick.

  keep the USB stick physically controlled)

**On the second laptop:**

```bash

cd ~
tar -xzvf /path/to/usb/silicritter-migration-2026-04-24.tar.gz

# 2. Clone the repo.
git clone git@github.com:edhodapp/silicritter.git
cd silicritter

# 3. Recreate the venv (won't transfer — Python/CUDA versions differ).
python3.12 -m venv .venv
source .venv/bin/activate
pip install -e .
pip install 'jax[cuda12]'   # adjust for the second laptop's CUDA/driver

# 4. Install the pre-commit hook (symlink to the shared script).
ln -sf ~/tools/code-review/pre-commit-hook.sh .git/hooks/pre-commit

# 5. Smoke test: tests should be 125 passed, 100% branch coverage.
.venv/bin/pytest tests/ -q

# 6. Verify the memory arrived.
ls ~/.claude/projects/-home-ed-silicritter/memory/MEMORY.md
```

If `~/.claude/` already exists on the second laptop with different
content, the `tar -xzvf` will overwrite. Review the extraction listing
before accepting — or extract into a tmp dir first and merge by hand
if the second laptop has its own global config worth preserving.
